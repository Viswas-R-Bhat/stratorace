Training Pipeline

End-to-end process from raw telemetry to a trained PPO agent ready for inference.

01
Data Ingestion
FastF1 API pulls all race sessions 2022–2024. Telemetry, weather, lap times, compound, tyre age loaded per session.
02
Cleaning & Filtering
Null lap times dropped. Safety-car laps (>2× median) removed. Outlier stint lengths excluded. Ergast fallback warnings suppressed.
03
Feature Engineering
LapTimeDelta = driver lap − personal best. TyreAge, compound encoding, stint number, gap calculations computed per lap.
04
Environment
Custom OpenAI Gym environment wraps each race. Each step = one lap. Observation = 9-dimensional state vector. Actions = {PIT, STAY}.
05
PPO Training
12,400 episodes across all races. Clip ratio ε=0.2, entropy coefficient 0.01, value loss coeff 0.5. Adam optimizer lr=3e-4.
06
Validation
15 unseen races held out. Agent evaluated on pit/stay decision accuracy, position delta, and time-saved vs actual team strategy.
Reward Function

The reward signal is the core design decision. StratoRace uses a composite reward that balances position gain, tyre degradation penalty, and pit timing.

Reward Formulation
R(t) = α · ΔPosition(t) − β · DegradationPenalty(TyreAge, Compound) − γ · PitCostTime [if action = PIT] + δ · PositionGainFromUndercut(t)
α · ΔPosition
α = 3.0
Primary reward — every position gained scores 3 points. Position lost = −3. The dominant signal.
β · DegPenalty
β = 0.8
Penalty proportional to tyre age beyond optimal window. Soft cliff: exponential after threshold lap.
γ · PitCostTime
γ = 0.4
Fixed pit stop time loss (~22s) normalised and weighted. Discourages unnecessary stops.
δ · UnderCut
δ = 1.5
Bonus when pitting leads to an undercut gain — passing a car that pits later on older tyres.

Limitations & Future Work

StratoRace is an academic prototype. Known limitations that future iterations should address.

⚠
No Live Telemetry
Current model is batch-trained, not live. Real-time inference requires a streaming FastF1 or official data source.
⚠
Binary Action Space
{PIT, STAY} only. Compound selection is post-hoc heuristic. Future: multi-discrete action space including compound choice.
⚠
No Multi-Agent
Each driver is modelled independently. Team strategy interactions (stacking, cover stops) are not yet captured.
→
Future: Bi-LSTM
Sequential tyre degradation modelling via Bi-LSTM as a feature extractor before the PPO network, if baselines prove insufficient.
